# Azure IoT Hub / PubNub ## Connecting devices, browsers and servers

-- Mikael Gueck, 13.11.2015

### Mission * build an IoT data stream app * build a general PubSub browser app * manage with minimal vendor lock-in * vendors come and go
### Azure IoT Hub * closely resembles the AWS offering * message queue and device/application state repo * AMQP connection * REST publish, but only device-to-cloud * Event Hub REST connections are PubSub, single delivery
### Azure IoT problems * feature list and documentation seems unreliable * products evolve so quickly that developing on them seems risky
### PubNub / Pusher * established players * message queue and device/application state repo * HTTP connection with keys in url * dev kits for every language * PubNub: $2.5 per one million messages * PubNub: minimum billing $150/month * Pusher: slightly different billing
### Winner * PubNub * long term, whoever supports the standard protocols best
