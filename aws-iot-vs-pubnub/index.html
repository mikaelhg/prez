# AWS IoT / PubNub ## Connecting devices, browsers and servers

-- Mikael Gueck, 28.10.2015

### Mission * build an IoT data stream app * build a general PubSub browser app * manage with minimal vendor lock-in * vendors come and go
### AWS IoT (beta) * beta, more features coming soon * message queue and device/application state repo * MQTT connection with a client SSL certificate * [no MQTT WebSocket support](https://forums.aws.amazon.com/thread.jspa?messageID=679681) yet * fake "REST" HTTP publish, but no subscribe * $5 per one million messages
### AWS integrations * MQTT → Kinesis firehose * MQTT → Lambda → MQTT * MQTT → Machine Learning * MQTT → DynamoDB
### AWS IoT problems * can't really subscribe from browser * no MQTT QoS 2
### PubNub / Pusher * established players * message queue and device/application state repo * HTTP connection with keys in url * dev kits for every language * PubNub: $2.5 per one million messages * PubNub: minimum billing $150/month * Pusher: slightly different billing
### Winner * PubNub - right this second * AWS IoT - after they browser features
